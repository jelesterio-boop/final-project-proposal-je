Description

In my application, the user attempts to beat the dealer by getting a count as close to 21 as possible, without going over 21. The game will be set up where the user has to respond to a preset set of questions posed by the dealer. For each card that is revealed to the user, the user has the option of hitting another card or not. In the end, the total sum that will be calculated based on his decisions.

Prioritized list of features that can be delivered in time

  • The cards that are revealed to the user should be randomized.
  • The sum of the integers from all card(s) must be calculated correctly.
  • Different responses should appear if the user scores above 21, exactly at 21, or lower than 21.

    Non-prioritized list of features that can be implemented if given more time

  • Graphics of the cards being dealt.
  • Graphics of the dealer expressing different emotions in response to the result of a given game.
